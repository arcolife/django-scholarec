Afterthought will be the tool to run (not predefined) jobs on *nix systems.


The Procedure:

As of now, It would:

  • Take an URI as argument
  • Fetch the data of the URI
  • Strip/parse the formatted data (with instruction sets)
  • Execute the commands/jobs.

By operation, one can simply consider it as a dynamic, lazy-loaded shell-script.

Current Situation

When running a script it follows the routines put in it at the creation time (which may have been obsoleted at the time of execution).


Proposed Improvements

Afterthought fetches the latest routines, which would be the latest route-to-follow for the operation - without the user needing to worry about it (or get the 'latest' script).

End Results:

This makes task automation and server/application deployment way easier, and takes much less effort to maintain.
